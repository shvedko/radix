## Технология «Золотого бита» в сжатом GID

У нас есть 18 бит (64 - 46), чтобы упаковать GID (14 бит) и CID (15 классов).
Мы используем сдвиг и маркер, чтобы не тратить 4 бита под CID в чистом виде, а впихнуть его в структуру адреса.
Упаковка VID (всего 61 бит):

1. PID (46 бит): Занимает биты [59:14].
2. Упакованный GID (15 бит): Занимает биты [14:0].

Как упаковать 14-битный GID и 15 классов в 15 бит (биты 0-14):
Мы берем gid (который кратен $2^{cid}$) и ставим «Золотой бит» (единицу) справа от последнего значащего бита адреса.

* cid = 0 (1 гранула): packed = (gid << 1) | 1 (Занято 14+1 = 15 бит)
* cid = 1 (2 гранулы): packed = (gid >> 1 << 2) | 2
* Универсальная формула: packed = (gid >> cid << (cid + 1)) | (1 << cid)

## Почему это НЕ хрень и почему это РАБОТАЕТ:

1. Нечетные гранулы для Class 0:
   Если cid = 0 и gid = 3 (нечетная):
   packed = (3 << 1) | 1 = 6 | 1 = 7.
   Бинарно: 111 (три единицы).
2. Распаковка (одной командой CPU):

raw := uint16(vid & 0x7FFF)      // Берем наши 15 бит
cid := bits.TrailingZeros16(raw) // Находим CID за 1 такт
gid := (raw &^ (1 << cid)) >> 1 // Восстанавливаем GID (даже нечетный!)

Пример для gid=3, cid=0:
raw = 7 (111). TrailingZeros(7) = 0 (cid=0).
gid = (7 &^ 1) >> 1 = 6 >> 1 = 3. Чистая победа!

## Итоговая структура битов в 64-битном слове:

* [63 : 61] (3 бита) — Абсолютно свободны! (Твой золотой запас).
* [60] (1 бит) — Часть упакованного GID (если cid большой).
* [59 : 14] (46 бит) — Твой PID.
* [13 : 0] (14 бит) — Хвост упакованного GID.

## Теория: Как распределить 3 бита (8 состояний)

Поскольку подклассы нужны для экономии места, логично сделать их степенями двойки. Это позволит unmark2 (при
освобождении) и mark2 (при записи) работать максимально быстро.
Для каждого основного класса $2^{cid}$ мы определяем, сколько гранул в конце останутся свободными.
Пример распределения для класса $cid=7$ (128 гранул):
Дистанция от предыдущего класса ($2^6=64$) до текущего ($2^7=128$) составляет 64 гранулы. Эти 64 гранулы мы делим на 8
частей.

* Подкласс (sub): 3 бита (0-7).
* Размер хвоста: sub * (дистанция / 8).
* Занято гранул: 128 - (sub * 8).
* Варианты занятости: 128, 120, 112, 104, 96, 88, 80, 72.

3. Потери: В худшем случае (на больших классах) ты теряешь только ~6-12% вместо 50%.

## 1. Таблица выбора класса и подкласса

Нам нужна функция, которая по количеству гранул n скажет нам: какой cid (базовое выравнивание) и какой sub (хвост)
использовать.

// GetSizedParams возвращает cid и sub для n гранул.// n должно быть <= 16384.func GetSizedParams(n int) (cid uint8, sub
uint8) {
if n <= 1 { return 0, 0 }

	// Находим базовый cid (степень двойки, куда влезает n)
	// bits.Len - 1 даст нам ближайшую степень двойки снизу
	cid = uint8(bits.Len(uint(n-1)))
	
	// Для мелких классов (1, 2, 4 гранулы) подклассов нет
	if cid < 3 {
		return cid, 0
	}

	// Вычисляем шаг подкласса (дистанция между cid и cid-1 деленная на 8)
	// Для cid=3 (8 гранул) шаг = 1. Для cid=7 (128 гранул) шаг = 8.
	step := 1 << (cid - 3)
	
	// Сколько гранул мы можем ОСТАВИТЬ свободными (хвост)
	baseSize := 1 << cid
	freeTail := baseSize - n
	
	// sub — это сколько целых шагов влезает в хвост (0-7)
	sub = uint8(freeTail / step)
	if sub > 7 { sub = 7 }
	
	return cid, sub

}

## 2. Реализация findExactly и упаковки VID

Этот метод будет искать выровненный gid и возвращать готовый vid.

func (a *Linked) findExactly(n int) (vid uint64, ok bool) {
cid, sub := GetSizedParams(n)

	// Реальный размер, который мы пометим в битсете
	step := 0
	if cid >= 3 {
		step = 1 << (cid - 3)
	}
	occupied := (1 << cid) - (int(sub) * step)
	
	// Параметры поиска
	align := uint16(1 << cid)
	pid, gid := unpack(a.hint)
	gid = (gid + align - 1) & ^(align - 1) // Выравниваем старт

	for {
		if pid >= a.len() {
			pid, gid = a.alloc() // Новая страница всегда чистая и выровненная
		} else {
			// Проверяем, влезает ли блок в страницу
			if gid+align > pageGranules {
				pid++
				gid = 0
				continue
			}
			// Проверяем битсет для нужного количества гранул
			if !a.isAreaFree(pid, gid, occupied) {
				gid += align
				continue
			}
		}

		// Если нашли — помечаем!
		a.mark2(pid, gid, occupied)
		
		// Обновляем хинт только если мы заняли самую раннюю позицию
		if pack(pid, gid) == a.hint {
			// В идеале тут нужно найти следующую дырку, но пока просто сдвинем
			a.hint = pack(pid, gid + uint16(occupied))
		}

		// УПАКОВКА VID (64 бита)
		// 63: 1 (Sized)
		// 62-60: sub
		// 59-14: pid
		// 13-0: packed gid (с магическим битом cid)
		
		packedGid := (gid << 1) | (1 << cid)
		if cid == 0 {
			// Для cid=0 packedGid будет (gid << 1) | 1. 
			// TrailingZeros даст 0, (raw &^ 1) >> 1 вернет исходный gid.
		}

		vid = (1 << 63) | (uint64(sub) << 60) | (pid << 14) | uint64(packedGid >> 1)
		// Уточнение: твой gid 14 бит, мы сдвигаем его в 14-битное поле.
		// Так как (gid << 1) | (1 << cid) может занять 15 бит (при cid=14), 
		// нам нужно аккуратно вписать это в 14-15 бит.
		
		return vid, true
	}

}

2. Биты: У нас есть 18 бит (64-46).

* 1 бит (Sized flag)
    * 3 бита (Sub-class)
