* Architecture: Гибрид Slab-allocator и Buddy-system. Иерархические битсеты (3 уровня) позволяют за $O(1)$ находить свободные страницы и блоки.
* Search Engine:
  * SWAR (SIMD Within A Register): Поиск внутри 64-битного слова без ветвлений через размазывание бит (mask |= mask >> s).
  * Alignment: Гарантированное выравнивание блоков по границе $2^n$ гранул, что делает «хвосты» блоков пригодными для меньших классов.
  * Complexity: Поиск свободного блока занимает ~10.7 нс, что сопоставимо с системными аллокаторами на C.
* Storage Magic:
  * Varint Headers: Использование put28/get28 (1–4 байта) для хранения реального размера данных в первой грануле.
  * Adaptive Tails: Классы размеров с шагом (step) позволяют возвращать неиспользуемые гранулы в систему, минимизируя внутреннюю фрагментацию.
* Performance:
  * Write: ~40 GB/s (на уровне пропускной способности кэша L1/L2).
  * Read: ~31 GB/s (благодаря «обману» курсора через rem > 0).
  * Zero-Copy/Zero-Alloc: Весь API работает на unsafe.Slice и прямых манипуляциях с памятью страниц.
